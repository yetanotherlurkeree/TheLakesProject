Vanderlande: 2016 - heden

Tijdens deze periode werkte ik aan verschillende projecten. Dit waren zowel interne als externe projecten.

Bij alle projecten werkte ik met Siemens PLC’s. De libraries waren geschreven in STL en heb ik hier veel ervaring en kennis mee opgedaan.

Daarnaast heb ik met verschillende versioning systemen gewerkt, zoals Synergy (IBM Tools Suite) en SVN, en ook change request systemen, zoals Change en Jira.

Bij de projecten waar ik op site ging werkte ik vaak samen met verschillende disciplines, zoals de electrische discipline en de mechanische discipline.

Project: DAA Dublin

Rol: Controls project engineer

Voor het vliegveld van Dublin werkte ik aan de software van twee systemen.

Ik hielp eerst met een kleine systeem. Dit systeem hebben onder grote druk in bedrijf kunnen stellen. Hierbij heb ik geholpen met het aanpassen van de software en was ik betrokken bij het maken van speciale software voor specifieke functionaliteiten van het vliegveld. Uiteindelijk hebben we dit getest met een Factory Acceptance Test (FAT). Dit werd daarna ook op site getest, waar ik ook bij betrokken was. Bij problemen met de software loste ik op dat moment op.

Ik praatte met zowel de klant als derde partijen, maar ook intern communiceerde ik met de installatie team als de team op kantoor.

Na een succesvolle Site Acceptance Test en een succesvolle Integrated Site Acceptance Test, bleef ik ook op site om te helpen, terwijl het systeem in operatie ging.

Daarna hielp ik mee met een grotere systeem, waarin ik vooral meehielp met het oplossen van problemen in de software en het uitvoeren van testen. Uiteindelijk heeft dit ook geleidt tot een succesvolle FAT voor dit systeem.

Project: Amazon BCN8 Sorting center

Rol: Controls project engineer

Dit project vindt plaats in Barcelona, Spanje. Hier was ik voornamelijk verantwoordelijk voor het in bedrijf stellen van één van de PLC’s in de sorteer centrum. Hierbij werkte ik samen met electricians bij het controleren van de IO. Daarnaast configureerde ik software op plekken waar het nodig was, zoals snelheden.

Ik heb hier ook gewerkt met de safety PLC, waarvan ik de noodstoppen testte. Hierdoor kreeg ik een idee hoe de safety software werkt.

Project: R&D Middle Bag Tubs

Rol: R&D Engineer[Interne project ICT]

Vanderlande besteedde een project uit aan ICT. Ik was hierbij een betrokken als een teamlid.

In dit project werd een maat van een tub groter gemaakt en dit moest ook gedaan worden in de software. Hierbij was ik verantwoordelijk voor een product van het systeem. Ik heb daarvoor software omgezet en de software getest.

Aan het einde van deze periode was ik ook verantwoordelijk voor het genereren van de software. Hierbij controleer ik of de software van de team te genereren is met de tooling van Vanderlande. Desnoods configureer ik de tooling met extra instellingen om de software te genereren.

Project: GTAA Toronto

Rol: Controls project engineer

Dit project vond plaats bij het Pearson International in Toronto in Terminal 3. Deze terminal werd vernieuwd en hierbij werd het baggage systeem ook vernieuwd.

Ik heb hier geholpen met het in bedrijf stellen van software op verschillende plekken in deze terminal. Zo was ik betrokken bij diverse checkin systemen en screening systemen.

Hierbij heb ik bij één screening systeem meegeholpen met het testen van de software voor IFAT. Hierbij presenteerde en voerde ik de testen uit voor de klanten.

Echter het grootste gedeelte van deze periode was vooral op site. Zo werkte ik samen met een team, waar we de systemen testen. Hier had ik vooral de rol om de problemen op te lossen die uit de testen kwamen. Ik heb hier ook een rol opgepakt om als support te dienen voor de operaties.

Zo onderzocht ik ook verschillende problemen in de bestaande systemen en werd ik weleens gebeld door operatie om een probleem op te lossen.

Bombardier: 2013 - 2016

Rol: Test engineer [Interne project ICT]

Voor dit project heb ik vooral test cases uitgevoerd en nieuwe testcases gemaakt voor nieuwe functionaliteiten van een bestaande treinenserie. Hierbij leerde ik om systeem testen te maken en nieuwe testcases te ontwerpen. Ik voerde deze testen dan ook uit op een trein, waarbij ik moest samenwerken met Duitse machinisten.

De machinisten praatten vooral Duits en de testcases waren ook in het Duits. Hierdoor kon en moest ik mijn Duits verbeteren. Hierdoor ging ik dus uiteindelijk beter Duits praten.

Na het uitvoeren van de testen maakte ik ook de testrapporten. Bij eventuele gefaalde tests hielp ik soms mee met het analyseren van het probleem in de software.
